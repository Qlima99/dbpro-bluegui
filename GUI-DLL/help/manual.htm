Installation

 

Note:  This assumes that you have installed DBPro to C:\Program Files\Dark Basic Software\Dark Basic Professional, if DBPro is installed to another folder, then you will need to use that path instead.

 

  • To install the plugin, you need to copy the gui2.dll file into your users plugins folder (C:\Program Files\Dark Basic Software\Dark Basic Professional\Compiler\plugins-user)
  • To make the plugins commands recognisable by the editor, you must be using the latest version (upgrades available from www.thegamecreators.com) and  you need to copy the gui2.ini keywords file into your keywords folder (C:\Program Files\Dark Basic Software\Dark Basic Professional\Editor\Keywords)
  • In order to be able to access the help files from the editor (by pressing F1 after typing a command name), you need to copy the gui2_help folder to C:\Program Files\Dark Basic Software\Dark Basic Professional\Help

 

Introduction

           

BlueGUI is a graphical toolkit, allowing you to incorporate various controls (buttons, edits, listboxes, toolbars, panels etc.) and dialogs (open, save, colour etc.) into your DBPro programs.  BlueGUI refers to these controls as gadgets. 

 

Some important BlueGUI Principles:

 

Activating the Plugin

 

At the top of any program which uses BlueGUI, you must call the startBlue command which sets up the plugin for use.  You must pass the UserID and Key which you were given to this command.  Please remember that your UserID and Key are individual to you.  If you wish to post any BlueGUI code on forums to get help or for the benefit of other users, please remove the UserID and Key from the code.

 

Handles

 

The commands which come with DBPro use index numbers to refer to different objects, images, sounds and so on.  For example, you might do make object cube 1,5 to create a cube.  You have to specify a number (1 in this case) and then you use that afterwards to refer to the object (eg: move object 1,5). 

 

BlueGUI is a little different, when you create a new gadget, menu, font etc. you dont specify a number.  Instead the command returns a handle to the new gadget / menu / font, for example:  popupMenu=createPopupMenu().

 

A handle is the place in your computers memory where the new object is stored, similar to the way that a shortcut on your desktop links to a file somewhere else on your PC.

 

Handles are normal integer variables.  They are no different from the other variables you use in your code.  If you want to make a handle accessible to the whole program use the global keyword (eg:  global popupMenu : popupMenu=createPopupMenu() )  At the time of writing, you cannot define the variable and create the gadget in the same statement.  global button=createButton(0,0,75,25,"THIS IS NOT OK",0,0) will not work, you need to put a colon in as shown above.

 

Important:  If you pass an invalid handle to a BlueGUI command your program will probably crash.  This could happen if you create a gadget and store its handle in a variable, then pass that handle to a GUI command after deleting the gadget.  In order to make sure your program is safe, reset the variable containing the handle to zero after you delete the gadget.  If you pass zero as a handle to any GUI command, it will be safely ignored.

 

deleteGadget button

button=0

 

 

Index numbers of lists, tabs etc.

 

Gadgets like listboxes use index numbers to refer to the items in them.  In BlueGUI, the first item always has an index of 0.  So if you wanted to remove the first item in a listbox you would do removeItem listboxHandle,0.

 

Constants

 

Alongside this plugin is a file called gui.dba.  In BlueGUI some commands have lots of different numbers with different meanings which they can return.  The gui.dba file puts helpful names to these numbers (eg: MOUSE_CLICK = 0x202), and you should include it with any project that uses BlueGUI v2.

 

Designing your program

 

Programmers all over the world have spent years arguing about how to lay out your code to make it easy to understand and manage.  If you are already an experienced programmer, you will probably have found a layout and conventions that suit you.  If not, I recommend this basic outline for your programs:

 

 

`Global variables to hold gadget handles

`eg:

global mainWin

global toolWin

 

`Call our function to create the user interface for the program

createUI()

 

`Main Loop

do

      getEvent

      `Process gadget events  do this before anything else

      processEvents()

 

      `Other program code goes here

      `. . .

loop

 

function createUI()

      `Code to create user interface goes here

endfunction

 

function processEvents()

      `Code to process gadget events goes here

endfunction

 

Other points to note are:

-         Give variables meaningful names (mainWin is a lot more helpful than mw)

-         Try and split everything up into separate functions as much as possible

 

The next step is to start creating some gadgets!

 

Creating Gadgets

 

To create a gadget, you use the plugins creation commands.  These create the new window with the position, size and other options you supply and return a handle to the new gadget.  A handle is BlueGUIs equivalent of a telephone number  a way of identifying each gadget, which you then pass to other gadget functions to set the properties of the gadget (such as size or position) and retrieve the properties of the gadget.  Here is an example of the steps to create a simple button (note that the syntax is createButton(x position, y position, width, height, caption, style, parent) )

 

newButton=createButton(0,0,75,25,OK,0,0)

 

This button is positioned at the top left hand corner of the main screen, is 75 pixels wide and 25 pixels high.  Its caption is OK, its a 3D button because the style parameter is 0. 

 

Note that newButton is simply a normal integer variable.  In DBPro there is no need to declare it, you can just write the code as above, however you will probably want to be able to access the button from anywhere in your code, its a good idea to make newButton a global variable.

 

The parent parameter is 0, so the gadget is created inside the main screen.  If you want to create a gadget inside of another gadget, you would set the owner gadget as the parent:

 

newPanel=createPanel(0,0,100,100,0)

newButton=createButton(0,0,75,25, OK,0,newPanel)

 

This creates the button inside the panel.  If you move the panel, the button moves as well.  If you hide the panel, the button is also hidden.  If you disable the panel, the button is also disabled.

 

You can resize / move the gadget after you have created it, but if the gadget is not going to change size/position during the program, you should set its position and size correctly in the create command.

 

When a new gadget is created, it is visible immediately, unless it is created with a parent gadget that is already hidden.

 

Unlike version 1 of BlueGUI, all gadgets work quite happily whether they are a child of another gadget or not.

 

Using Gadget Commands

 

Once you have created a new gadget, you can use the handle returned with the other commands to manipulate the gadget.  Some commands work on all gadget types, other commands only work on a specific gadget type or a group of gadget types.  For example, the positionGadget command will work on any gadget, but addItem will only work on gadgets such as listboxes and comboboxes.

 

Eg:

 

newButton=createButton(0,0,75,25, OK,0,0)

positionGadget newButton,100,100

 

Please see the Command Reference, Examples and Sample Applications section at the end of this guide for information on getting help for each command used with BlueGUI.

 

 

Gadgets Available In BlueGUI

 

BlueGUI supplies the following gadgets for use in your programs:

 

  • Buttons  these are very simple gadgets which give a pushed in appearance when clicked and have a caption on them.  They are used to instruct your program to perform a task such as hiding an object.  Buttons can also display pictures.
  • Labels  simple pieces of text which are used to label other gadgets so users know what they do.  They are also used to display status information to the user.  Labels can also be used to display pictures, either from a file or from an image in memory (eg: a 3D realtime viewport)
  • Panels  simple grey rectangles used to contain other gadgets. 
  • Checkboxes  A box with a caption attached.  The user can check/uncheck the box by clicking on it and provide simple yes/no information for the program (eg:  a Match Case checkbox on a Find Text dialog box)
  • Radiobuttons  Similar to checkboxes, except that instead of just yes/no, by creating several radiobuttons with the same parent, you can give the user a range of options (eg: Up or Down search direction in a Find Text dialog box)  they use circles rather than boxes.
  • Frames  Very similar to panels, except that they have a thin frame around them with a caption in the top left-hand corner.  They are used to group gadgets visually, for example in a Print dialog, all the options relating to the number of pages to be printed are contained in a Page Range frame.
  • Listboxes  A white rectangle which displays a list of selectable items.  This is used to display datasets (eg: languages which your program can use) to the user.
  • Comboboxes  Similar to listboxes, except that the items are hidden away most of the time and you click on a drop down arrow to reveal the list, which then folds up when you select a new item.  The virtue of this is that less screen space is used.  As only one item (the currently selected one), the purpose of a combobox is to get a selection from the user, rather than just displaying a list.
  • Edits  These are boxes which allow the user to enter & edit text.  These are used when you want to get text-based information such as a name from the user.  They can also be used to display the contents of text files on screen.  As they are scrollable, you can give the user access to a lot of text in a fairly small screen space.
  • Tabs  Often there may be too many settings or options in a dialog or panel to be able to fit on screen.  In this case you use tabs to divide up your dialog / panel into sections.  For example in a level editor your options panel might have tabs for Object,World and Lights.  Tabs have a grey rectangular display area where you place other gadgets, and dividers at the top.
  • Statusbars  These are grey bars which appear at the bottom of the screen and display important status information or hints to the user.  For example in a text editor the statusbar would display the current Line and Column of the cursor.  In a world editor, a description could be shown when you mouse over objects that can be added to a scene.  Statusbars are usually divided up into separate panels for each piece of information you want to display.
  • Toolbars -  These are bars at the top of the screen with icon buttons in that provide quick access to the most important/commonly used commands in your program such as Open, Save, Cut, Copy, Paste.  Toolbar buttons are flat and the icons are usually partially transparent (the transparent colour is taken from the pixel at the top-left hand corner of the icon).  When you position the mouse over a button it becomes 3D and then flat when the mouse moves out.  Toolbar buttons can also be toggle on/off buttons, such as the Bold, Italic and Underline buttons in a word processor
  • Scrollbars  These are light grey bars with arrows at either end, and a slider or thumb.  By moving the slider either by clicking and dragging or by clicking on the arrow buttons, the user can scroll the contents of the workspace or dialog.  Scrollbars have adjustable ranges and thumb sizes.  They can be horizontal or vertical.
  • UpDown Gadgets  These are a simple pair of up & down arrows used to change an integer value.  When you click the up / down arrow, the value associated with it increases/decreases.  The rate of change increases the longer you hold down the button.  These gadgets are often attached to other gadgets (buddy gadgets) such as an Edit box so the user can see the value.  If the UpDown gadget is attached to another gadget, it will set its caption automatically.  UpDown gadgets can have a variable range.
  • Canvas Gadgets  These are empty gadgets which you can draw on using the various drawing commands (setDrawingColor, drawRect, drawLine, drawEllipse, drawPoint).  These are extremely useful for a wide variety of tasks, such as creating your own gadgets if you need something which isnt provided or if you want a non-windows graphical style.  They can also be used to display any other visual information you can think of. 
  • ListView Gadgets  These are used to display items in a list, but whereas listboxes only have text, ListView gadgets can have images and additional sub-items as well.  Examples of usage would include a texture browser or an online multiplayer server list.
  • TreeView Gadgets  These display items of text (with optional images) in an organised tree-like structure.  A good example of this would be the Browse For Folder dialog, which uses a tree-view to show all the folders in the system and how they are related to one another.
  • Window Gadgets  These are gadgets with an optional titlebar which are used to contain and organise other gadgets.  For example a paint program normally has a Tool Options window containing buttons, boxes and edit gadgets to change various properties.
  • Menus  These include the main menus which you see at the top of program windows containing options such as File, Edit, Help etc.  and also the popup menus which you see when you right click on something important in many programs.
  • Trackbar Gadgets  These are similar to scrollbars except that they have pointed sliders and little tick marks at regular intervals along the sides of the gadget.  These are used in the volume control dialog for example.
  • Web Browser Gadgets  These allow you to integrate Internet Explorer into your application to allow the user to browse websites, help files etc. 

 

See the help files for a full reference of the commands for creating and manipulating the different gadget types.

 

Responding to User Interaction (Mouse Clicks, Key Presses etc.)

 

In order to respond to user interaction such as mouse clicks or key presses, you have to look at the messages which Windows sends to a window when the user does something with it.  These messages are referred to by BlueGUI as events

 

To do this, you place the getEvent command at the top of your main loop.  getEvent gets information from Windows about the last thing that happened to the window, such as a mouse click or movement.  getEvent does not return any information, you use the eventSource, eventType, eventData and eventDataEx commands to get information about the event which occurred.

 

eventSource is the handle to the gadget to which the event happened.  This is the same handle returned by the create<gadgettype> command when you first created the gadget.

 

eventType is the kind of event which happened.  This is often a number which you will probably find hard to memorize, and looking it up all the time would be tiresome.  Hence youll find a file called gui.dba which came with the plugin.  This has a whole host of constants (values which do not change), which put helpful names to these numbers, for example MOUSE_CLICK (0x202) or MOUSE_MOVE (0x200).  You need to include this file in your project (on the Files tab in the normal editor or Includes tab on the Project tab in BlueIDE).  After you have included the file in your project, you can simply write if eventType()=MOUSE_CLICK in your main program instead of if eventType()=0x202.  This is much easier to understand when you come back to it later.

 

For the help files and examples to work you must include the gui.dba constants file in your project.

 

eventData and eventDataEx return additional information about the event which occurred.  For example with a MOUSE_MOVE event eventData returns the mouse buttons which were being pressed at the time.

 

A simple program loop which responds to a click on a button by displaying the good ole Hello World message would be written like this:

 

Button=createButton(0,0,75,25, OK,0,0)

 

Do

   getEvent

  

  if eventSource()=Button

     if eventType()=MOUSE_CLICK

        message Hello World!

     endif

  endif

Loop

 

For the lazy people out there or die-hard BlueGUI v1 users, there is also a gadgetClicked() command which returns the handle to the last gadget clicked since the last call to gadgetClicked()

 

Button=createButton(0,0,75,25, OK,0,0)

 

Do

    Current=gadgetClicked()

    If current=Button

        Message Hello World!

    Endif

Loop

 

Note that this is far less flexible than the event system and does not return any other events or information about the event, but may be convenient in very small programs.

 

I suggest that you set up a small project now using the above code (dont forget to include gui.dba) and marvel at your first BlueGUI v2 program J

 

Performance and SYNC RATE

 

Usually in an application, you will only process one event per program loop.  If your program loop is running fairly fast this will not be a problem because you will be able to catch all the events that happen.  However, if you have limited the speed of your program loop using SYNC RATE (usually used to ensure a constant framerate on all PCs), and a large number of events occur at once then your program will need several loops to catch up, resulting in a slight delay between the user performing an action and your program responding.  In order to deal with this, you can place another loop within the main loop, keep calling getEvent until eventType() returns 0.

 

sync on : sync rate 10

 

panel=createPanel(200,200,200,200,0)

 

do

     repeat

         getEvent

        

         if eventSource()=panel and eventType()=0x200

               print n

               inc n

         endif

     until eventType()=0

     sync

loop

 

In the above example, move the mouse around on the panel, the numbers will stop incrementing very soon after you stop moving the mouse.  However, rem out the line repeat and the line until eventType()=0 and you will see that the numbers will continue incrementing for quite a while after you stop moving the mouse. 

 

As the number of events that can be remembered by the plugin before being unloaded is limited, it is possible to lose events if too many occur in between calls to getEvent.

 

Catching events that happen to the main window

 

If you wish to catch events such as mouse movements that occur to the main window, you must convert the main window into a gadget.  To do this, use the mainWindow and createGadgetFromWindow commands, after which you can treat the resulting gadget like any other (although attempting to reposition / resize using gadget commands is not recommended)

 

Eg:

mainWinGadget=createGadgetFromWindow(mainWindow())

 

Creating and Working With Menus

 

Users of BlueMenus version 1.0 should be aware that menus work very differently under BlueGUI v2, however the new system is more flexible and generally requires less code.

 

There are 3 basic types of menu which a normal program uses:

-         Main Menu:  This appears at the top of your program window and has options such as File, Edit, View and Help

-         Sub Menu:  These appear underneath the main menu.  For example the one attached to the File item of the main menu would have options such as Open, Save and Exit.  Each sub menu item can contain a number of sub menus underneath it.

-         Popup Menu:  Also known as the context menu, this pops up when you right-click on a certain area of your program.

 

The steps you need to setup a menu are:

-         Create a main menu, and add items to it

-         Create the various sub menus, and add items to them

-         Write code to handle menu item clicks

 

To create the menu, use the createMenu() command for a main menu, createSubMenu() for the sub menus, and createPopupMenu() for popup menus.  Each of these commands returns a handle to the new menu.  Menus are initially empty.

 

After that you can use the same commands to add / remove / change items from any of the 3 types of menu.  To add a new item to a menu, use addMenuItem.  When you add a menu item using addMenuItem, you specify an ID for it.  The ID can be any number.  When you click on a menu item, eventData() will contain this ID so you know which item was clicked, and when you use other commands, you will need to pass this ID to refer to the menu item.

 

To check for menu selections, call getEvent at the top of your program, and then use eventType to check for a MENU_CLICK message (note that the events come from the parent gadget, not the menu itself).  Finally eventData is used to check the ID of the item which was clicked.  Menus are modal, this means that whilst the user is making a selection from the menu, your program code will pause.

 

This example shows how to create a simple program menu (make sure you include the gui.dba constants file!)

 

global mainMenu

global fileMenu

global helpMenu

 

`Create a new main menu

mainMenu=createMenu(0)

 

`Add items to the main menu, specifying the values

`we want them to return when they are selected

addMenuItem mainMenu,File,1

addMenuItem mainMenu,Help,2

 

`Create the two submenus

fileMenu=createSubMenu(mainMenu,1)

helpMenu=createSubMenu(mainMenu,2)

 

`Add items to the submenus

addMenuItem fileMenu,Exit,3

addMenuItem helpMenu,About,4

 

`Main Loop

do

      getEvent

     

      if eventType()=MENU_CLICK

            select eventData()

                  case 3 : `Exit

                        end

                  endcase

                  case 4 : `About

                        message Simple Menus Demo

                  endcase

            endselect

      endif

loop

 

Firstly we create global variables to store the menu handles.  We then use createMenu to create a new main menu, which is attached to the main window because the parent parameter is 0.  File and Help items are then added with IDs of 1 and 2 respectively. 

Next two submenus are created using createSubMenu.  In each case we specify the handle of the parent menu, and the item on that parent menu which we want the sub menu to appear under.  Items are then added to each of the sub menus using the addMenuItem command.

Finally, we create our main program loop.  getEvent is used to retrieve information about menu events, and we then check for a MENU_CLICK event.  If we receive a MENU_CLICK message, we then check eventData to see which item was selected, and the program is either ended or a help message is shown.

 

Dialogs

 

Dialogs are another important part of the user interface.  BlueGUI supports the following dialog types:

 

-         Message Boxes

-         Colour Selection Dialog

-         Open/Save Dialog

 

Message Boxes

 

Message boxes are used to display important information to the user.  They are modal which means that the program will pause until the user has accepted the dialog.  To display a simple message use the message, warningMessage or errorMessage commands.

 

message This is a simple message

warningMessage Something went wrong

errorMessage Something went seriously wrong!

 

If you want a simple Yes / No response to a question such as Are you sure you want to exit?, then use the questionMessage command.  questionMessage returns 1 if the user selected Yes, or 0 if they selected no.

 

if questionMessage(Do you want to exit?,Exit?)

      end

endif

 

Colour Selection Dialog

 

The colour dialog allows users to pick a colour visually.  To create a colour dialog, use the colorDialog command.  Again, this is a modal dialog.  colorDialog returns a normal RGB (Red, Green, Blue) colour which you can then use with other DBPro commands.  Colour dialogs have a custom colours box which remembers the colours stored in it until the program closes.

 

Open / Save Dialog

 

Open / Save dialogs allow the user to pick a file to open or enter the name to save a file to.  The commands are openDialog and saveDialog.  Both commands effectively work in the same way.  The only difference is the caption of the button is Open for an open dialog, or Save for a save dialog.  openDialog displays a standard open dialog with the specified caption and filetypes, and returns the filename of the file which the user selected or an empty string if no file was selected.

 

The syntax for openDialog is:

 

filename$=openDialog(caption$, fileTypes$)

 

caption$ is the text which appears at the top of the dialog, eg: Open File

 

fileTypes$ specifies the types of files which the user can select.  The basic format is

 

Description (*.extension)|*.extension|Description (*.extension)|*.extension

 

The description appears in the File Types combo box.  When the user selects a type, the dialog filters out all files which do not match the specified extension.  Each part is separated by a pipe ( | ) character.  If you want to display all files, use *.* as the extension.

 

Putting it all together, here is the code for a simple pair of open & save dialogs which allows the user to select text files:

 

openName$=openDialog("Open File","Text Documents (*.txt)|*.txt")

message openName$

saveName$=saveDialog("Save File","Text Documents (*.txt)|*.txt")

message saveName$

 

 

Fonts

 

To change the font used by a gadget to draw its text, you first create a new font with the properties you want using createFont.  This returns a handle to the new font, and you can then use applyFont to set the font for a gadget.

 

The syntax for createFont is:

 

fontHandle=createFont(name$,pointSize,bold,italic,underline)

 

name$ can be any font on the users system.  It is best not to use an obscure / non-standard font because the user may not have it on their system, and the font will not be created.  You can use PERFORM CHECKLIST FOR FONTS to get information about the fonts on the users computer.

 

You can apply the same font to as many different gadgets as you wish.  The syntax for applyFont is:

 

applyFont gadgetHandle,fontHandle

 

To create a large, bold, underlined Verdana font and apply it to a button, the code would be:

 

button=createButton(0,0,200,200,"OK",0,0)

bigFont=createFont("Verdana",72,1,0,1)

applyFont button,bigFont

 

Be aware that redrawing takes longer with very large fonts, which can lead to flickering.

 

Note that not all gadget fonts can be controlled, and not all fonts support a complete range of styles or sizes.  For example bitmap fonts only come in a small range of sizes.

 

Using Rainbow

 

Rainbow is a new feature in BlueGUI v2.  It is a visual user interface designer, which allows you to click and place gadgets, set their properties, and then export DarkBASIC code for use in your program.  Trying to work out positions and sizes of gadgets in code is a tedious and time-consuming process.  Using Rainbow you can do this very quickly and easily.

 

Rainbow is written in DBPro using BlueGUI, and comes complete with the source code.

 

To start the program, run the Rainbow.exe file in the Rainbow folder.

Creating and Arranging Gadgets

 

To create gadgets, select the type you want to create using the toolbar at the top, and then click in the main drawing area to create the gadget.  If you click on an existing gadget in the drawing area, the new gadget will be a child of the gadget you clicked on. 

 

This has many uses.  For example, if you wanted to create a side panel for a 3D level editor, you would first create a tab gadget.  You would then create several panels as children of the tab gadget (eg for Object, Light and Sound).  Finally you would place controls on each of those panels (eg on the light panel there would be a Choose Colour button).  When your program was running, you would hide or show each panel depending on which tab was selected. 

 

Important Note:  For tab gadgets, you must click on the TAB CAPTION (the word tab which appears at the top of the gadget in the editor) in order to select the gadget, or create a new gadget as a child of it. 

 

To position or resize a gadget, first select it by clicking on it, or by clicking its icon in the layout tree.  You will then see four black markers appear at the corners of the gadget.  Click in the middle of the gadget and drag to position it, or click and drag the black markers to resize it.

 

Window gadgets are created as children of the main DBPro window, if you want to change this, you must edit the exported code.

 

There are various keyboard commands you can use to make manipulating & arranging gadgets easier:

 

-         Press V when a gadget is selected to hide/show the gadget in the editor.  This does not affect the exported code, but it may be convenient in the editor.

-         Press L when a gadget is selected to lock/unlock the gadget in place.  This prevents / enables movement and resizing of the gadget

-         Press the Delete key when a gadget is selected to delete the gadget and any of its children.

 

Setting Gadget Properties

 

The most important property of a gadget to set is its name.  When you export the code to create the user interface, a global variable will be created to hold each gadgets handle.  You then use this global variable to refer to the gadget in other parts of your code.  The name of the global variable will be whatever you set the name property to.

 

To set the name, select the gadget in the editor.  Click on the Name property in the Property Tree, and then change the name in the edit box which will become enabled.  Press Enter or click on the main form when you have changed the name.

 

You edit other properties such as Text, position and alignment in the same way.  Not all properties are reflected in the editor.  For example Initially Enabled and Initially Visible have no effect in the editor, but they will result in changes to the code which is produced.

 

Saving & Loading Layouts

 

Use the File menu to load and save gadget layouts for editing later on.  Layouts are saved to .gui files which are simple text files that describe the layout of the gadgets.  Remember to save regularly!

 

Exporting DarkBASIC Code

 

Go to File > Export DarkBASIC Code and you will be prompted for a filename, Rainbow will then create a .dba file with the code needed to create the layout.  The code consists of a number of global variables (one per gadget), and a function called createUI().

 

Add the .dba file as an include to your project (I recommend using the #include command instead of including it via the editors files list  otherwise if the source file is open in the editor and you export a new version from Rainbow, the DarkBASIC editor will use the source from the open DBA file, rather than the version on disk).  After adding the .dba file to your project, call createUI() at the start of your program to create the layout.

 

For example, this is the code for a simple layout created by Rainbow, a file called interface.dba:

 

`User Interface Setup Code

`Generated by Rainbow version Beta 2

`===========================================

`Call createUI() to setup the user interface

`-------------------------------------------

 

`Global Variable Declarations for Gadgets

`----------------------------------------

global mainTabs

global optionsPanel

global btnSelectFill

global btnSelectTex

global btnHMap

global lbBrusShape

global cmbBrusShape

global lbBrusBlend

global cmbBrusBlend

global frameBrusStyle

global pnlBrusStyle

global rdLower

global rdSmooth

global cbUseFill

global rdCarve

global cbUseTex

 

`Create User Interface

`---------------------

function createUI()

    If Not GUI_CONSTANTS Then errorMessage "GUI Constants not defined.  Please include gui.dba when you compile" : end

   

    mainTabs=createTabs(0,300,640,147,0)

    optionsPanel=createPanel(0,25,640,117,mainTabs)

    btnSelectFill=createButton(230,13,115,25,"Select Fill Colour",0,optionsPanel)

    btnSelectTex=createButton(230,43,115,25,"Select Texture",0,optionsPanel)

    btnHMap=createButton(230,73,115,25,"Select Heightmap",0,optionsPanel)

    lbBrusShape=createLabel(367,8,120,15,"Select a brush shape:",optionsPanel)

    cmbBrusShape=createComboBox(367,28,225,100,optionsPanel)

    lbBrusBlend=createLabel(367,58,135,15,"Brush blend mode:",optionsPanel)

    cmbBrusBlend=createComboBox(367,78,225,100,optionsPanel)

    frameBrusStyle=createFrame(12,7,200,95,"Brush Style",optionsPanel)

    pnlBrusStyle=createPanel(4,14,185,70,frameBrusStyle)

    rdLower=createRadioButton(2,22,75,20,"Lower",pnlBrusStyle)

    rdSmooth=createRadioButton(2,42,75,20,"Smooth",pnlBrusStyle)

    cbUseFill=createCheckBox(87,2,75,20,"Use Fill",pnlBrusStyle)

    rdCarve=createRadioButton(0,0,0,0,"",frameBrusStyle)

    cbUseTex=createCheckBox(0,0,0,0,"",frameBrusStyle)

   

endfunction

 

The code in our main source file would be:

 

#include interface.dba

 

createUI()

 

do

   getEvent

   `code to process events goes here

loop

 

Note that in order to use Rainbows exported code, you must include the gui.dba constants file in your project.

 

After exporting code from Rainbow, the program will ask you whether or not you want to copy the code to the clipboard.  If you prefer to copy and paste the code into the DarkBASIC editor, then click yes.  Then right-click and select Paste in the DB editor.

 

Rainbows purpose as an editor is a time-saver.  You cannot set all the various properties and options from within Rainbow.  In order to do this you may need to edit the createUI function after exporting the code.  This applies to window gadgets in particular.  Rainbows exported code will feature a normal resizeable window.  If you want a toolwindow or a fixed window (no minimize / maximize buttons), you will need to edit the exported code.

 

Using Project Creator

 

Project creator is a small utility which makes it easy to create BlueGUI-based DarkBASIC projects.  Project creator will set up all the necessary project files (including the gui.dba file) and optionally include the XP themes file and the web browser library (gui2_web.dll).  Depending on the template you select, it will also write out a basic framework for your code.

 

Project creator can be found in the Utilities folder of the examples.  Enter the name and select the location where you want your project to be created, select the template to use from the drop down box and whether or not you want XP themes and web browser gadgets to be enabled then click the Create Project button.

 

Notes:

Before attempting to run the newly created project, remember to change the UserID and Key parameters of the startBlue command to the ones you were provided with when you purchased the plugin. 

 

When you create the project, the executable name will be set to project.exe, if you decide to change it, you will need to rename the .manifest (XP themes) file to <new exe name>.exe.manifest

 

 

Command Reference, Examples and Sample Applications

 

To access the complete command reference which explains the syntax and usage of each command, plus the various examples and sample applications which come with BlueGUI, run the BlueGUI Help Viewer application from the Start Menu.

 

The examples are simple demos explaining how to use different gadgets, and you should refer to them first.  The sample applications show how to put all the components together to get a complete application.
